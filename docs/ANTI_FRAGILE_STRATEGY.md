## 一、策略概述

### 1.1 核心理念

| 事实 | 推论 |
|------|------|
| 散户亏损率 70-90% | 反向操作有统计优势 |
| 脆弱地址代表"韭菜共识" | 他们的极端行为是反向信号 |
| 清算瀑布前必有仓位聚集 | 可以预判清算猎杀机会 |

**核心公式：**
```
Alpha = 聪明钱正向跟单 + 脆弱地址反向操作

其中：
- 聪明钱跟单 → 跟随正确的人
- 脆弱地址反向 → 成为收割者而非被收割
```

### 1.2 策略配比

| 策略 | 占比 | 理由 |
|------|------|------|
| **聪明钱正向跟单** | 30-40% | 有 alpha 但样本有限 |
| **脆弱地址反向** | 50-60% | 样本量大，统计优势强 |
| **现金/观望** | 10-20% | 两边都不明确时 |

**为什么脆弱反向占比更高：**
1. 脆弱地址数量远多于聪明钱（10:1 甚至更多）
2. 统计样本更大，信号更稳定
3. 亏损是市场的常态，反向操作有天然优势

## 三、地址分类模型

### 3.1 聪明钱评分模型

```
总分 = 盈利能力(40分) + 风控能力(30分) + 稳定性(30分)

【盈利能力 40分】
├── 胜率 (15分)
│   ├── 50% → 5分
│   ├── 55% → 10分
│   └── 60%+ → 15分
├── 盈亏比 (15分)
│   ├── 1.0 → 5分
│   ├── 1.5 → 10分
│   └── 2.0+ → 15分
└── 总PnL% (10分)
    ├── 5% → 3分
    ├── 10% → 6分
    └── 20%+ → 10分

【风控能力 30分】
├── 最大回撤 (15分)
│   ├── <20% → 15分
│   ├── <30% → 10分
│   ├── <40% → 5分
│   └── >40% → 0分
├── 实际杠杆 (10分)
│   ├── <2x → 10分
│   ├── <3x → 7分
│   ├── <5x → 4分
│   └── >5x → 0分
└── 止损使用率 (5分)
    ├── >50% → 5分
    ├── >30% → 3分
    └── <30% → 0分

【稳定性 30分】
├── 盈利分布 (10分): 无单笔>总盈利50% → 10分
├── 交易频率稳定性 (10分): 方差小 → 10分
└── 存活时长 (10分)
    ├── >90天 → 10分
    ├── >60天 → 7分
    └── >30天 → 4分

入池门槛: ≥65分 且 脆弱评分<40
```

### 3.2 脆弱地址评分模型

```
总分 = 风险行为(40分) + 亏损特征(35分) + 心态特征(25分)

【风险行为 40分】
├── 实际杠杆 (15分)
│   ├── >5x → 15分
│   ├── >3x → 10分
│   └── >2x → 5分
├── 仓位利用率 (15分)
│   ├── >80% → 15分
│   ├── >60% → 10分
│   └── >40% → 5分
└── 单币种集中度 (10分)
    ├── >70% → 10分
    └── >50% → 5分

【亏损特征 35分】
├── 胜率 (15分)
│   ├── <35% → 15分
│   ├── <40% → 10分
│   └── <45% → 5分
├── 清算次数/月 (10分)
│   ├── ≥3 → 10分
│   ├── ≥2 → 7分
│   └── ≥1 → 4分
└── 总PnL% (10分)
    ├── <-20% → 10分
    ├── <-10% → 7分
    └── <0% → 4分

【心态特征 25分】
├── 浮亏加仓率 (10分)
│   ├── >60% → 10分
│   ├── >40% → 6分
│   └── >20% → 3分
├── 死扛指数 (10分): 亏损持仓/盈利持仓时间
│   ├── >3 → 10分
│   └── >2 → 6分
└── 追涨杀跌率 (5分)
    ├── >50% → 5分
    └── >30% → 3分

入池门槛: ≥55分
高危脆弱: ≥70分
```

### 3.3 分类决策树

```
分类优先级规则：

1. 先算脆弱评分
   └── if 脆弱评分 ≥ 70 → 直接标记为"高危脆弱"（不管聪明钱评分）

2. 再算聪明钱评分
   └── if 聪明钱评分 ≥ 75 且 脆弱评分 < 40 → 标记为"聪明钱"

3. 中间地带
   └── 聪明钱评分 65-75 或 脆弱评分 40-55 → 标记为"观察区"

4. 其他 → 中性地址（不关注）
```

**分类矩阵：**

| | 脆弱评分 <40 | 脆弱评分 40-55 | 脆弱评分 ≥55 |
|--|-------------|---------------|-------------|
| **聪明≥75** | 🟢 聪明钱 | ⚠️ 观察 | ❌ 异常(排除) |
| **聪明65-75** | ⚠️ 观察 | ⚠️ 观察 | 🔴 脆弱 |
| **聪明<65** | 中性 | 🔴 脆弱 | 🔴 高危脆弱 |

### 3.4 脆弱地址等级分类

| 等级 | 定义 | 反向价值 |
|------|------|---------|
| **L1 轻度脆弱** | 胜率 40-45%，偶尔爆仓 | 低 - 可能只是运气差 |
| **L2 中度脆弱** | 胜率 30-40%，月爆2次 | 中 - 有反向参考价值 |
| **L3 重度脆弱** | 胜率 <30%，高杠杆死扛 | 高 - 优质反向信号源 |
| **L4 极度脆弱** | 连续亏损，频繁FOMO | 极高 - 最佳反向指标 |

## 五、信号生成逻辑

### 5.1 脆弱地址反向信号触发条件

| 条件 | 阈值 | 操作 | 信号强度 |
|------|------|------|---------|
| 脆弱地址多空比极端 | >75% 同向 | 反向开仓 | ⭐⭐⭐ |
| 脆弱仓位集中在某价位 | 清算价聚集度 >30% | 狙击清算 | ⭐⭐⭐⭐ |
| 脆弱地址大幅加仓 | 24h 加仓 >50% | 反向准备 | ⭐⭐ |
| 脆弱+聪明钱方向相反 | 方向分歧度 >0.6 | 跟聪明钱 | ⭐⭐⭐⭐⭐ |
| 脆弱地址集体止损 | 止损触发率 >20% | 顺势跟随 | ⭐⭐ |

### 5.2 跟单时机规则

| 场景 | 是否跟单 | 原因 |
|------|---------|------|
| **首次开仓** | ✅ 跟 | 明确的方向信号 |
| **同向加仓（浮盈<20%，小比例）** | ✅ 跟 | 金字塔加仓 |
| **同向加仓（浮盈>30%，大比例）** | ❌ 不跟 | 可能追高 |
| **同向加仓（浮亏<5%，支撑位）** | ⚠️ 谨慎跟 | 看是否预设计划 |
| **同向加仓（浮亏>10%）** | ❌ 不跟 | 死扛行为 |
| **减仓** | ✅ 同比例减 | 跟随止盈/止损 |
| **平仓** | ✅ 同步平 | 信号结束 |

### 5.3 信号整合公式

```
最终信号 = 
    0.4 × 聪明钱正向信号 
  + 0.5 × 脆弱地址反向信号 (加权)
  + 0.1 × 市场结构信号 (清算、FR等)
```

## 七、特征计算方法

### 7.1 交易周期定义

```
一次完整交易 = 从开仓到完全平仓的所有 fills

识别规则：
1. 按 (address, coin) 分组
2. 累计 szi 变化，从 0 → 非0 → 0 为一个周期
3. 周期内的 closedPnl 累加 = 该交易的最终盈亏
```

### 7.2 核心特征计算

```python
def calculate_all_features(address, fills, state):
    features = {}
    
    # 从 fills 计算
    trades = group_into_trades(fills)
    winning_trades = [t for t in trades if t.pnl > 0]
    losing_trades = [t for t in trades if t.pnl < 0]
    
    features['win_rate'] = len(winning_trades) / len(trades)
    features['profit_loss_ratio'] = avg_profit(winning_trades) / avg_loss(losing_trades)
    features['avg_win_holding'] = avg_holding_time(winning_trades)
    features['avg_loss_holding'] = avg_holding_time(losing_trades)
    features['loss_add_ratio'] = count_loss_adds(fills) / count_all_adds(fills)
    features['liquidation_count'] = count_liquidations(fills)
    
    # 从 state 计算
    features['actual_leverage'] = state.totalNtlPos / state.accountValue
    features['margin_utilization'] = state.marginUsed / state.accountValue
    features['coin_concentration'] = max_coin_position(state) / state.totalNtlPos
    
    return features
```

### 7.3 边界情况处理

```
【新地址处理】
- 交易<10笔：不评估
- 交易10-20笔：只做脆弱判定（更容易识别）
- 交易>20笔：完整评估

【单币种专精地址】
- 不扣分，但标记 coin_specialist=True
- 跟单时只跟该币种的信号
- 单地址对该币种敞口上限提高到 5%（普通 2%）

【极端市场处理】
检测条件：BTC 24h 跌幅 > 15%
- 暂停所有脆弱/聪明评估更新
- 记录为"极端市场日"，这天的数据不计入统计
- 恢复后用极端市场前的数据继续计算
```

## 八、回测验证方案

1. 聪明钱池的平均收益是否显著高于市场？
2. 脆弱地址池的反向操作是否盈利？
3. 分类准确率是多少？

```
Step 1: 历史数据准备（假设有90天数据）
- Day 1-30: 训练期，用于初次分类
- Day 31-90: 验证期，观察分类后的表现

Step 2: 聪明钱验证
- 在 Day 30 筛选出的聪明钱池

- 计算：
  - 池子平均收益 vs BTC 收益
  - 跟单模拟收益（扣除滑点）
  - 多少人仍保持聪明钱标准

Step 3: 脆弱地址验证
- 在 Day 30 筛选出的脆弱池
- 跟踪 Day 31-90 的表现
- 计算：
  - 池子平均亏损率
  - 反向操作模拟收益
  - 多少人继续亏损

Step 4: 分类稳定性验证
- 每7天重新评估分类
- 计算分类变动率
- 目标：聪明钱周分类稳定率 > 80%
```
