# QuantWorld 需求文档

## 1. 项目目标

构建一个基于多智能体技术的交易市场仿真系统，用于：
- 预测市场情绪演化
- 模拟各类市场参与者行为
- 生成交易策略参考

---

## 2. 核心功能

### 2.1 多智能体仿真

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 智能体创建 | 根据配置创建不同类型的智能体 | P0 |
| 行为模拟 | 各智能体根据人设和环境做出决策 | P0 |
| 交互通信 | 智能体之间可以交互、影响 | P0 |
| 记忆系统 | 智能体具有短期/长期记忆 | P1 |

### 2.2 市场模拟

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 订单簿 | 模拟买卖订单簿 | P0 |
| 价格计算 | 根据供需计算价格变动 | P0 |
| 成交撮合 | 简化的订单撮合逻辑 | P1 |
| 流动性模拟 | 模拟市场流动性 | P2 |

### 2.3 数据接入

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 实时行情 | 接入交易所 API 获取实时价格 | P0 |
| 历史数据 | 历史 K 线数据 | P1 |
| 新闻/舆情 | 获取市场新闻作为输入 | P1 |
| 链上数据 | 巨鲸动向、交易所流入流出 | P2 |

### 2.4 输出报告

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 情绪报告 | 当前市场情绪评估 | P0 |
| 预测报告 | 短期价格走势预测 | P0 |
| 仿真回放 | 查看仿真过程的详细记录 | P1 |

## 3. 智能体角色设计

### 3.1 核心角色（P0）

#### 散户（Retail Trader）
- **数量**：3-5 个
- **类型**：
  - 乐观型：倾向于看多，FOMO 心理强
  - 中性型：根据市场信号决策
  - 恐慌型：风险厌恶，容易割肉
- **行为特征**：
  - 情绪化决策
  - 参考 KOL 意见
  - 追涨杀跌
- **决策依据**：
  - 社交媒体情绪
  - 价格涨跌
  - KOL 喊单

#### 量化交易员（Quant Trader）
- **数量**：1-2 个
- **行为特征**：
  - 纯数据驱动
  - 不受情绪影响
  - 高频交易
- **决策依据**：
  - 技术指标（MA、RSI、MACD）
  - 价格突破信号
  - 套利机会

#### 巨鲸（Whale）
- **数量**：1 个
- **行为特征**：
  - 大额持仓
  - 能影响市场价格
  - 有明确的吸筹/出货周期
- **决策依据**：
  - 长期趋势判断
  - 散户情绪（反向指标）
  - 资金成本

#### KOL/分析师（Opinion Leader）
- **数量**：1-2 个
- **行为特征**：
  - 发布市场分析
  - 影响散户情绪
  - 可能有利益导向
- **决策依据**：
  - 技术分析
  - 宏观消息
  - 粉丝互动

#### 新闻源（News Feed）
- **数量**：1 个
- **行为特征**：
  - 注入外部事件
  - 触发市场反应
- **输出**：
  - 利好消息
  - 利空消息
  - 中性消息

### 3.2 扩展角色（P1）

#### 机构交易员（Institutional Trader）
- 大资金、看基本面
- 根据研报和宏观数据决策

#### 做市商（Market Maker）
- 提供流动性
- 维护盘口

#### 链上监控者（On-chain Watcher）
- 追踪巨鲸动向
- 提供链上数据分析

## 4. 仿真流程

### 4.1 初始化阶段
1. 加载配置（币种、智能体数量、初始资金）
2. 创建智能体实例
3. 初始化市场状态
4. 获取初始行情数据

### 4.2 仿真循环
```
每个时间步（如 1 分钟）：
  1. 获取最新行情数据
  2. 注入外部事件（如新闻）
  3. 各智能体观察环境
  4. 各智能体做出决策（买入/卖出/持有）
  5. 执行交易，更新订单簿
  6. 计算新的市场价格
  7. 更新智能体状态和记忆
  8. 记录日志
```

### 4.3 输出阶段
1. 生成市场情绪报告
2. 生成价格预测
3. 保存仿真日志

## 6. MVP 范围

### Phase 1: 最小可行产品（2 周）

**目标**：5 个智能体模拟 BTC 市场情绪演化

**包含**：
- [x] 基础智能体框架（散户 × 3 + KOL × 1 + 新闻源 × 1）
- [ ] 简化的价格模型（不含完整订单簿）
- [ ] 接入 OKX 实时价格
- [ ] 命令行输出仿真结果
- [ ] 基础情绪报告

**不包含**：
- 完整订单簿撮合
- 前端界面
- 链上数据
- 多币种

### Phase 2: 功能增强（2 周）

- 增加量化交易员、巨鲸角色
- 简化订单簿
- 智能体记忆系统
- 仿真回放功能

### Phase 3: 完整版（4 周）

- 多币种支持
- 前端界面
- 链上数据接入
- 历史回测功能

## 7. 成本预估

### 7.1 LLM API 成本

**MVP 阶段（5 个智能体）**：
- 每轮仿真：5 个智能体 × 1000 tokens ≈ 5000 tokens
- 每日仿真：24 小时 × 60 轮 = 1440 轮
- 每日 tokens：1440 × 5000 = 720 万 tokens
- 使用 Qwen-Plus（$0.4/M tokens）：约 $3/天

**优化后**：
- 批量处理，减少调用次数
- 使用更小的模型处理简单决策
- 预计：$1-2/天

### 7.2 服务器成本

- 轻量级 VPS：$20-50/月

### 7.3 月度总成本

- MVP：$50-100/月
- 完整版：$100-300/月
